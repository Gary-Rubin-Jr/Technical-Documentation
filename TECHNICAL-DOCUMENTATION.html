INTRODUCTION

The CSS Grid Layout Module is intended for the creation of responsive layouts based on a grid coordinate system. Its design allows for the organization of content into both columns and rows. Using a grid layout, child elements of the grid container are positioned by assigning their location to specific grid lines. This method grants developers precise control over their layouts in a simple and manageable way.

The scope of this page is to introduce the basic principles of this layout method. I strongly recommend that you follow up with the linked content in the resources section.

    Key Terms

  • Grid Container :
    The mothership… this is the element to which display: grid; is applied. Some powerful bits of code.

  • Grid Item :
    This is a child of the grid container. It is important to note that grid items can also be grid containers unto themselves and contain child items of their own. Yes, grids within grids is a thing.

  • Grid Line :
    These lines give the grid structure. A grid can have horizontal row lines and vertical column lines. As many or as few of these lines can be used and they can be of symmetric or asymmetric size. In other words, you have complete control over the size and shape of your grid lines.

  • Grid Track :
    An entire row or column along one of the grids axis.

  • Grid Cell :
    An individual block on the grid. Grid items can be assigned to individual cells or to larger groups of multiple cells.

  • Grid Area :
    A section of the grid which contains multiple cells. Grid areas can be defined and named allowing for the assignments of grid items to that area.

Now that the essential terms have been defined we can continue to explore the implementation of this layout method. Through this document you will have the opportunity to manipulate the example code. This code is found at the bottom of the stylesheet and is identified with comments.

GRID DEFINITION

Unless our element is defined as a grid container, all we have is a block display. Which is not much to work with.

Take the following sample code:
<div class="grid_container grid_container_1">
<div class="example_item example_item_1">1</div>
<div class="example_item example_item_2">2</div>
<div class="example_item example_item_3">3</div>
<div class="example_item example_item_4">4</div>
</div>

We define the element’s display property by assigning it the value of grid or inline-grid.

/*==================DECLARE GRID DISPLAY HERE===========*/
.grid_container_2 {display:grid;}

Lets also add some color to make it easier to see what is going on.
Feel free to change these colors as you like, we will be looking at them for a bit. Look for this comment:

Because the columns and rows of our grid container have not been yet been defined our display grid statement results in all available space given to the grid container to be evenly distributed amongst the child items. Four rows have been created within the parent container and each child assigned to one row.

Next we will define this container as having three rows and three columns and each being one fraction of the available space in track size. There are a few methods by which this can be done, in this example the method used to define the rows means the same as the method used to define the columns.
Take some time to experiment with different values but be sure to set them back to the original values before moving on.

/*=================Define Rows and Columns==============*/
.grid_container_3{
grid-template-rows: repeat(3, 1fr);
grid-template-columns: 1fr 1fr 1fr;}

ITEM POSITIONING

Positioning items within a grid layout is very simple. This is after all the principal appeal to this layout module. All that is required is to state the beginning and ending grid lines for each axis. In this example, our container has three rows and three columns. This makes the grid line values range from one to four. Lines one and four are the outer edges of the container. Lines two and three are the internal dividing lines. Inspecting this element with Firefox will show the gridlines with their assigned number.

/*=================Position Items=============*/
.grid_container_5 .example_item{
height:100%;
width:100%;}
.grid_container_5 .example_item_1{
grid-row: 1/2;
grid-column: 1/2;}
.grid_container_5 .example_item_2{
grid-row: 2/3;
grid-column: 1/2;}
.grid_container_5 .example_item_3{
grid-row: 1/3;
grid-column: 2/4 ;}
.grid_container_5 .example_item_4{
grid-row:3/4 ;
grid-column:1 /4 ;}

ALIGN & JUSTIFY

Align and Justify Items

Alignment and justification of items refers to how grid items are displayed within their corresponding cells. Alignment affects positioning within the vertical axis while justification affects the horizontal axis. To these properties we can assign values that will determine how the content is displayed within the grid. These values are:

  • auto
  • normal
  • start
  • end
  • center
  • stretch
  • baseline
  • first-baseline
/*===========Align & Justify Items===========*/
.grid_container_6{
align-items: start;
justify-items: end;}

.grid_container_6 .example_item{
height:90%;
width:90%;
display: grid;
place-items: center center;}/*this could work with just one value*/

Align and Justify content

Alignment and justification of content comes into play when our grid area is smaller than the grid container that holds it. For this example, the grid has been set to occupy 75% of the grid container. As before, alignment and justification each affect a certain axis. However, in this case the properties deal with how the grid itself is distributed within the available space. Use developer tools to inspect this grid container and see how the grid lines are repositioned as you change the values assigned to the alignment and justification properties. Experiment with combinations of the following values:

  • start
  • end
  • center
  • space-between
  • space-around
  • space-evenly
/*===========Align & Justify Content===========*/
.grid_container_7{
grid-template-rows: repeat(3, 25%);
grid-template-columns: repeat(3, 25%);
align-content: space-around;
justify-content: space-evenly;}
